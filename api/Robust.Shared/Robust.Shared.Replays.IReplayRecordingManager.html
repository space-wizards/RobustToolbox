RecordReplayMessage(Object)

Queues some net-serializable data to be saved by a replay recording. Does nothing if IsRecording is false.

Declaration
void RecordReplayMessage(object obj)
Parameters
Type Name Description
System.Object obj
Remarks

The queued object is typically something like an EntityEventArgs, so that replays can simulate receiving networked messages. However, this can really be any serializable data and could be used for saving server-exclusive data like power net or atmos pipe-net data for replaying. Alternatively, those could also just use networked component states on entities that are in null space and hidden from all players (but still saved to replays).

This gets invoked whenever a replay recording is starting. Subscribers can use this to add extra yaml data to the recording's metadata file, as well as to provide serializable messages that get replayed when the replay is initially loaded. E.g., this should contain networked events that would get sent to a newly connected client.
