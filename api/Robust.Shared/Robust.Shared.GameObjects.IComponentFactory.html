Interface IComponentFactory

Handles the registration and spawning of components.

Namespace: Robust.Shared.GameObjects
Assembly: Robust.Shared.dll
Syntax
public interface IComponentFactory
Remarks

When referring to component names, this is the name the component has been registered as, and what's used in prototypes. However, most commonly the type is referred through by an interface.

Before a component can be spawned, it must be registered so things such as name, networking ID, type, etc... are known to the factory. Components are registered into a registry. The relevant methods for writing to this registry are . The data is exposed for reading through GetRegistration(String, Boolean) and its overloads. This data is returned in the form of a , which represents one component's registration.
