AppearanceComponent

The appearance component allows game logic to be more detached from the actual visuals of an entity such as 2D sprites, 3D, particles, lights... It does this with a "data" system. Basically, code writes data to the component, and the component will use prototype-based configuration to change the actual visuals. The data works using a simple key/value system. It is recommended to use enum keys to prevent errors. Visualization works client side with derivatives of the VisualizerSystem class and corresponding components.
